📖 快速操作指南

🖼 替换场景样张

打开“样张管理”，搜索并点选场景，然后点击“选择新样张”。
系统会统一转换为高质量 JPEG，并按场景 ID 保存。整个操作只能从本机完成。

➕ 添加新场景

点击“新增场景”从空白开始，或者在相似场景旁点击“复制”再修改。
表单保存后，顶部会提示存在未保存修改；点击“保存到项目”即可自动分片、规范化和检查。

🏷 添加新 Tag

1. 在「Tag 管理」标签页点击「+ 新增 Tag」
2. 输入英文名（用于 prompt）、中文名（用于显示）、分类
3. 系统会自动分配 ID
4. 导出 JSON 覆盖 data/tags.json

🔄 保存与自动备份

“保存到项目”会先在 runtime/maintenance-backups 创建备份，再写入场景源文件并自动检查。
如果检查失败，旧文件会自动恢复。你也可以随时点击“导出备份”额外保存一份 JSON。

⚠️ 注意事项

• 场景 ID 必须唯一，格式为 sc001、sc002…
• Tag 英文名必须唯一，用于 prompt 生成
• char 字段决定角色 LoRA：nene → ayachi_nene_v11，natsume → shiki_natsume_v11
• dual_pose_verified 标签表示该双人场景有 OpenPose 骨架图
• “下架”不会立即写入文件，只有点击“保存到项目”后才生效
• 直接保存和替换样张仅允许在本机入口执行，朋友分享链接不能修改项目

编辑场景

用一句能快速看懂画面的标题，后续在场景库里更容易找到。
建议写清角色、地点、动作和情绪；提示词会以这里为故事底稿。
