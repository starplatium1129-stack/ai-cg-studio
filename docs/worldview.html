🌍 项目定位

这份文档不讲实现细节。
只说明一件事：这个个人工具想解决什么，又明确不做什么。

希望打开网站时，先看到一个值得画下来的瞬间，

再像整理 Galgame CG 分镜一样，

从故事、情绪、镜头和光影开始思考，

让每一张图片都成为一个有温度的场景。

— 绫季绘境 · 个人场景创作台

一、问题：为什么 AI 图"少点什么"？

你试过吗？花了半小时拼 Prompt，加了十几个标签，出来的图—
清晰、锐利、"好看"。

但总觉得少了点什么。

不是 Prompt 不够长。不是细节不够多。
而是画面里没有一个人在等你。

"没有导演思维的 AI 创作，只是堆标签。"

二、另一个世界：Galgame 是怎么创作的？

一张 Galgame CG，从来不是原画师一个人决定的。
它是编剧、导演、原画、摄影、色彩五个角色共同决定的。

导演说："放学以后，她在校门口等你。"
色彩说："夕阳，暖调，风。"
摄影说："逆光，回头，浅景深。"

然后原画师才开始动笔。

最后那张 CG 放在游戏里，玩家看到的不是标签，
是一个故事、一段回忆、一种情绪。

三、现状：AI 创作者面对的世界

现在 AI 创作者面对的是什么？

1girl, smile, school uniform, sunset, masterpiece, best quality...

一串标签。冷冰冰。没有故事。

想画的是“放学后她在樱花树下等人”，
却只能输入"1girl, cherry blossom, waiting, school"。

AI 能听懂标签，但听不懂故事。
— 工具要做的，就是把这个瞬间翻译成可生成的画面条件。

四、这个工具要做的事

它更像一张分镜工作纸：

先看到的不是一堆标签，
而是自己脑海里的画面。

写下“我想画一张她放学后等人的 CG”后，
再把故事整理成：

情绪（期待、温柔）
→ 镜头（远景、逆光、回头）
→ 色彩（夕阳、暖调、樱花色）
→ 构图（三分法、人物居中偏左）
→ Prompt（一串串精致的标签）
→ 最终画面

Prompt 不必成为第一步。
想细调时，它仍然可以查看和修改。

"Prompt 只是语言，Scene 才是画面。"

五、这个工具不准备变成什么

它不是一个 Prompt 收藏夹。
它不是一个标签管理工具。
它也不准备变成需要运营的公开绘图平台。

它是AI Galgame CG 创作工作台：

让自己和朋友可以像整理分镜一样思考，
少一点重复堆标签，多一点对故事和画面的控制。

六、希望达到的状态

使用时不必先说：

"我不会写 Prompt。"

而是可以先说：

"我想画一个放学后在樱花树下等人的宁宁。"

然后由工具准备一个可靠起点，细节再根据实际出图调整。

七、创作取向

后续增加功能时，优先保持这三条：

RULE 01
Scene First

先整理 Scene，再组合 Prompt。

RULE 02
Character First

人物是画面中心，背景与特效服务于角色。

RULE 03
Emotion First

比起标签本身，更在意画面留下的情绪。

"从故事，到画面。"
From Story to Scene.

八、维护边界

这是一个自用为主、偶尔分享朋友的项目。
账号、社区、商店、订阅和长期云端服务都不在当前计划中。

新功能应先证明它能改善实际创作，
并且不会让启动、安全和维护成本失去控制。

保持本地优先、分享可停止、内容可维护，
比追求看起来完整的平台功能更重要。

九、一句话总结

Prompt 描述图片。Scene 描述瞬间。

一张优秀的 CG，不是因为 Prompt 很长而优秀，
而是因为它捕捉到了一个值得记住的瞬间。

这个工具不负责把项目写得宏大，只负责让想画的瞬间更容易落地。
