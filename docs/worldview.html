🌍 世界观 (Worldview)

这份文档不讲技术，不讲功能。
只回答一个问题：我们为什么做这件事？

希望 AI 创作者不再面对一堆冰冷的标签，

而是能够像 Galgame 原画师一样，

从故事、情绪、镜头和光影开始思考，

让每一张图片都成为一个有温度的场景。

— AI CG Studio · Scene Library for AI Artists

一、问题：为什么 AI 图"少点什么"？

你试过吗？花了半小时拼 Prompt，加了十几个标签，出来的图—
清晰、锐利、"好看"。

但总觉得少了点什么。

不是 Prompt 不够长。不是细节不够多。
而是画面里没有一个人在等你。

"没有导演思维的 AI 创作，只是堆标签。"

二、另一个世界：Galgame 是怎么创作的？

一张 Galgame CG，从来不是原画师一个人决定的。
它是编剧、导演、原画、摄影、色彩五个角色共同决定的。

导演说："放学以后，她在校门口等你。"
色彩说："夕阳，暖调，风。"
摄影说："逆光，回头，浅景深。"

然后原画师才开始动笔。

最后那张 CG 放在游戏里，玩家看到的不是标签，
是一个故事、一段回忆、一种情绪。

三、现状：AI 创作者面对的世界

现在 AI 创作者面对的是什么？

1girl, smile, school uniform, sunset, masterpiece, best quality...

一串标签。冷冰冰。没有故事。

用户想画的是"放学后她在樱花树下等人"，
却只能输入"1girl, cherry blossom, waiting, school"。

AI 能听懂标签，但听不懂故事。
— 除非我们帮它翻译。

四、我们想做的事

我们想做一面镜子：

让用户看到的不是一堆标签，
而是自己脑海里的画面。

用户说"我想画一张她放学后等人的 CG"，
系统帮他把故事翻译成：

情绪（期待、温柔）
→ 镜头（远景、逆光、回头）
→ 色彩（夕阳、暖调、樱花色）
→ 构图（三分法、人物居中偏左）
→ Prompt（一串串精致的标签）
→ 最终画面

用户永远不需要直接面对标签。
系统负责翻译。

"Prompt 只是语言，Scene 才是画面。"

五、这个工具不属于谁

它不是一个 Prompt 收藏夹。
它不是一个标签管理工具。
它不是另一个"AI 绘画网站"。

它是AI Galgame CG 创作工作台：

帮助创作者像游戏原画师一样思考，
不像AI 使用者一样堆 Prompt。

六、最终目标

这个工具不是让用户说：

"我不会写 Prompt。"

而是让用户说：

"我想画一个放学后在樱花树下等人的宁宁。"

然后剩下的工作，工具替他完成。

七、产品原则

以后无论项目多大，增加多少功能，都不能违反这三条：

RULE 01
Scene First

先有 Scene，后有 Prompt。Prompt 永远是最后一步。

RULE 02
Character First

人物永远是画面中心。任何元素都服务于角色。

RULE 03
Emotion First

用户记住的不是 Prompt，而是画面带来的回忆。

"从故事，到画面。"
From Story to Scene.

八、给未来的话

如果有一天，这个项目不再是一个人维护，
而是有其他人参与开发—

请记住这份文档。
它会成为整个项目最重要的方向指引。

很多成熟的产品都有自己的设计哲学。
这份世界观，就是这个项目的灵魂。

九、两句话说清我们

Prompt 描述图片。Scene 描述瞬间。

一张优秀的 CG，不是因为 Prompt 很长而优秀，
而是因为它捕捉到了一个值得记住的瞬间。

别人: 5000 条 Prompt。
我们: 500 个 Scene。

完全不是一个东西。
