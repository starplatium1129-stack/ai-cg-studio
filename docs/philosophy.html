💡 创作哲学 (Philosophy)

所有功能设计的原点。每个决策回到这里对齐。

第零原则（最高）

Prompt 描述图片。Scene 描述瞬间。

这是整个项目的根基。一张优秀的 CG，不是因为 Prompt 很长而优秀，而是因为它捕捉到了一个值得记住的瞬间。

用户不是在"写 Prompt"，而是在"导演一个 Scene"。Prompt 自动系统生成，用户甚至不需要知道。

第零原则补充

Scene Engineering > Prompt Engineering

AI 圈几乎所有工具都在解决 "Prompt 怎么写？"。
没人在解决 "画面怎么设计？"

这是两个完全不同的问题。训练好的 LoRA 也需要回答：
用什么场景测试？ 用什么镜头测试？ 用什么光照测试？ 怎样判断 LoRA 是否稳定？
这些都超出了 Prompt 的范畴，属于 Scene。

进化链：Prompt Engineering → Scene Engineering → Visual Story Engineering

第一原则

先有画面，后有 Prompt

Prompt 永远是最后一步。脑海里的画面 → 故事 → 情绪 → 镜头 → 构图 → Prompt。

❌ 错误

Prompt → 生成 → 希望好看

✅ 正确

脑海画面 → 故事 → 情绪 → 镜头 → 构图 → Prompt

第二原则

一张 CG 必须讲一个故事

没有故事的 CG 只是一个美少女站在那里。有了故事，她才是"放学后在教室等你的那个人"。

❌ 空洞

女孩站着。女孩笑着。

✅ 有故事

放学以后，她一直在教室等你。她看见你以后，终于露出了笑容。

第三原则

人物永远是第一主角

人物 ★★★★★ · 背景 ★★★☆☆ · 特效 ★★☆☆☆。任何元素都不能抢人物。

背景服务人物。不是人物服务背景。

第四原则

每个元素都要有理由

为什么是夕阳？故事发生在放学。为什么是樱花？这是春天。为什么回头？她在等你。

Prompt 不应该出现无意义标签。

Scene ≠ Prompt ≠ 标签

❌ 不要这样想

站着、笑、学校、校服、夕阳

✅ 应该这样想

放学以后，她站在校门口一直等你，风吹起头发，夕阳照在她脸上

场景 > 标签

不是"教室"

是"放学后的教室"

不是"卧室"

是"睡前的一句晚安"

不是"神社"

是"新年的第一次祈福"

不是"海边"

是"第一次一起看海"

质量 > 数量

别人：5000 Prompt
我们：500 Scene

每一条都有故事、情绪、镜头、构图、推荐参数。这就是质量。
