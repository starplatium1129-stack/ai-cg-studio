💡 创作取向

记录这个工具怎样把故事、画面和 Prompt 放进同一条创作流程。

第零原则（最高）

Prompt 描述图片。Scene 描述瞬间。

这是整个项目的根基。一张优秀的 CG，不是因为 Prompt 很长而优秀，而是因为它捕捉到了一个值得记住的瞬间。

使用时先整理 Scene，再让系统组合 Prompt；需要时仍然可以查看、复制和调整 Prompt。

第零原则补充

Scene Engineering > Prompt Engineering

很多工具擅长解决 “Prompt 怎么写？”。
这个项目更关心 “画面怎么设计？”

这是两个完全不同的问题。训练好的 LoRA 也需要回答：
用什么场景测试？ 用什么镜头测试？ 用什么光照测试？ 怎样判断 LoRA 是否稳定？
这些都超出了 Prompt 的范畴，属于 Scene。

工作方式：故事 → Scene → Prompt → 画面

第一原则

先有画面，后有 Prompt

先把脑海里的画面拆成故事、情绪、镜头和构图，再整理 Prompt。

❌ 错误

Prompt → 生成 → 希望好看

✅ 正确

脑海画面 → 故事 → 情绪 → 镜头 → 构图 → Prompt

第二原则

让一张 CG 有一个故事

没有故事的 CG 只是一个美少女站在那里。有了故事，她才是"放学后在教室等你的那个人"。

❌ 空洞

女孩站着。女孩笑着。

✅ 有故事

放学以后，她一直在教室等你。她看见你以后，终于露出了笑容。

第三原则

人物优先于背景和特效

人物 ★★★★★ · 背景 ★★★☆☆ · 特效 ★★☆☆☆。任何元素都不能抢人物。

背景服务人物。不是人物服务背景。

第四原则

每个元素都要有理由

为什么是夕阳？故事发生在放学。为什么是樱花？这是春天。为什么回头？她在等你。

Prompt 不应该出现无意义标签。

导演模式创作流程

9 步完成一张 CG，像导演一样思考。

Scene ≠ Prompt ≠ 标签

❌ 不要这样想

站着、笑、学校、校服、夕阳

✅ 应该这样想

放学以后，她站在校门口一直等你，风吹起头发，夕阳照在她脸上

场景 > 标签

不是"教室"

是"放学后的教室"

不是"卧室"

是"睡前的一句晚安"

不是"神社"

是"新年的第一次祈福"

不是"海边"

是"第一次一起看海"

质量 > 数量

284 个 Scene 不是为了凑数量。

每一条都应有明确的故事、情绪、镜头、构图和可用参数；重复或不稳定的场景继续修改，不急着增加新编号。

适合的使用方式

🎨 从一个故事瞬间开始创作 Galgame 风格 CG
🖼 用固定场景测试宁宁与夏目的角色 LoRA
✨ 保存稳定组合，之后可以继续生成或比较变体
