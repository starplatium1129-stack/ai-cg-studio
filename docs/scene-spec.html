🎞 Scene 规范 (Scene Specification)

Prompt 以后不要编号。Scene 才是编号单位。

📋 SC0001 模板 — 官方 Scene 必填字段

Scene ID
SC' + 四位数字 (SC0001, SC0002...)
标题
中文 Scene 名，有温度 (例: 放学后的回眸)
故事
一句话故事描述。谁在什么地方、在做什么、为什么。
情绪
核心情绪 (恋爱、期待、害羞...)
推荐角色
ayachi_nene / shiki_natsume / 通用
Resolution
8 档, 带 VRAM 要求 (详见下方速查)
推荐 LoRA 权重
0.75 (默认) / 范围 0.6–0.8
Positive
遵循 Prompt Spec 的 9 模块顺序。每 Scene 唯一变量。
Negative
基础层 (通用 17 词) + 场景特定层 (3–5 词)
锁定参数 🔒 永远不动
CFG 5 · Sampler DPM++ 2M SDE · Steps 28 · Hires denoise 0.45
标签
用于 Scene 库检索的标签

🔒 锁定参数 永远不动

设一次, 保存, 以后再也不会改。每 Scene 唯一变的只有 Positive + Negative。

参数值唯一例外
CFG5 (永远)暗调场景 (月夜/夜灯/暗室) → 5.5
SamplerDPM++ 2M SDE (永远)无
Steps28 (永远)无
Hires Denoise0.45 (永远)追求更锐利 → 0.40
Hires Steps12–15 (永远)无
LoRA 权重0.75 (默认)过拟合 → 降到 0.6; 不够明显 → 升到 0.85

🛡 Negative 速查

📌 基础层 (永远带, 17 词)

任何 Scene 都不改。直接复制。

lowres, bad anatomy, bad hands, text, error, missing fingers, extra digit, fewer digits, cropped, worst quality, low quality, normal quality, jpeg artifacts, signature, watermark, username, blurry, artist name

📌 场景特定层 (每场景选 3–5 词叠加到基础层)

按场景类型查表。对立的东西不能同时出现就加。
