Version 1.5

Every Great CG Begins with a Story.

每一张优秀的 CG，都始于一个故事。

前言（Preface）

AI 绘画的发展，让越来越多的人能够创造属于自己的作品。

然而，大多数 AI 工具的发展方向，都围绕着 Prompt（提示词） 展开。

创作者不断学习新的标签。

不断记忆新的参数。

不断寻找更长、更复杂的 Prompt。

但是，我们认为：

Prompt 从来都不是创作本身。

Prompt 只是 AI 理解世界的语言。

而不是创作者表达世界的语言。

真正的创作，应该从脑海中的画面开始。

从故事开始。

从情绪开始。

从角色开始。

最后，Prompt 只是将这一切翻译成 AI 能理解的语言。

因此，我们希望创造一种全新的 AI 创作方式。

不是 Prompt Engineering。

而是：

Scene Engineering.

第一章
Vision（项目愿景）
我们为什么做这个产品？

目前几乎所有 Prompt 工具，都在帮助用户：

写 Prompt。

但是用户真正的问题并不是：

Prompt 怎么写？

真正的问题是：

我想画什么？

当用户脑海中已经出现一个画面的时候。

Prompt 本应是最后一步。

而不是第一步。

因此。

AI CG Studio 希望重新定义 AI 创作流程。

我们相信：

故事

↓

Scene

↓

Prompt

↓

CG

而不是

Prompt

↓

图片

第二章
我们想解决什么问题？

目前 AI 创作者主要存在五个问题。

第一

不知道画什么。

打开 WebUI。

面对空白 Prompt。

没有灵感。

第二

不知道 Prompt 怎么写。

不断复制。

不断修改。

不断试错。

第三

不会设计画面。

知道标签。

不知道构图。

不知道镜头。

不知道为什么别人画得好。

第四

LoRA 测试没有标准。

今天一个 Prompt。

明天另一个 Prompt。

无法横向比较。

第五

Prompt 越来越长。

越来越复杂。

越来越难维护。

最后自己都看不懂。

因此。

我们认为：

Prompt 不应该成为创作中心。

第三章
产品理念（Product Philosophy）

AI CG Studio 只有三个原则。

Rule One
Story First

先有故事。

再有画面。

最后才有 Prompt。

故事决定：

人物。

动作。

时间。

地点。

情绪。

镜头。

光照。

Rule Two
Character First

人物永远是第一主角。

背景服务人物。

光照服务人物。

构图服务人物。

所有元素。

都围绕人物展开。

Rule Three
Emotion First

用户记住的不是 Prompt。

而是：

那个黄昏。

那场烟花。

那次回眸。

那句晚安。

✅ 等你赴约的下午

Scene 永远应该讲一个故事。

Chapter 2.2 用户真正的问题

我们总结了五个核心问题。

Problem 1

不知道画什么。

不是不会写 Prompt。

而是不知道：

今天该画哪一张 CG。

Problem 2

不会构图。

例如：

知道：

教室

不知道：

应该坐窗边。

还是站门口。

还是趴桌子。

还是回头。
Problem 3

不会讲故事。

Prompt 网站：

告诉你：

school

但是没人告诉你：

为什么会来到学校。

Problem 4

不会建立统一工作流。

今天：

PromptA。

明天：

PromptB。

永远无法复现。

Problem 5

LoRA 测试混乱。

不同 Prompt。

不同参数。

无法比较。

LoRA 档案：每个 LoRA 的训练来源、推荐权重、适合的 Scene、测试记录。
经验沉淀：哪些组合稳定、哪些容易崩、哪些适合壁纸、哪些适合头像。

随着使用时间增长，这个库会越来越有价值。它不是简单收藏 Prompt，而是在积累创作经验。
